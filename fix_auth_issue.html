Issue: Your frontend is making API calls without proper authentication.

Error: "Not authorized to access this route"

Cause: Missing or invalid Authorization token in the request headers.

Step 1: Check Current Authentication Status

Step 2: Login to Get Valid Token

Step 3: Test API Call with Token

Step 4: Fix Your Frontend Code

Once authentication works, update your frontend code to always include the Authorization header:

// ‚úÖ CORRECT - Include Authorization header
fetch('/api/v1/exams?page=1&limit=10', {
  headers: {
    'Authorization': `Bearer ${localStorage.getItem('token')}`,
    'Content-Type': 'application/json'
  }
})

// ‚ùå WRONG - Missing Authorization header (causes 401 error)
fetch('/api/v1/exams?page=1&limit=10', {
  headers: {
    'Content-Type': 'application/json'
  }
})
